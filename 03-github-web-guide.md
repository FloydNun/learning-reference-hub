## Common Tasks on GitHub. com

### 1. Creating a New Repository
1. Click green **"New"** button (or + in top right)
2. Name your repository
3. Choose Public or Private
4. ‚úÖ Check "Add a README file"
5. Click **"Create repository"**

### 2. Creating an Issue (Task/Bug Report)
1. Go to **Issues** tab
2. Click **"New issue"**
3. Add a title (short description)
4. Add details in the description
5. Optional: Add labels, assignees
6. Click **"Submit new issue"**

### 3. Creating a Pull Request (PR)
1. Push your branch to GitHub (see Sheet 2)
2. Go to **Pull Requests** tab
3. Click **"New pull request"**
4. Select: base:  main ‚Üê compare: your-branch-name
5. Click **"Create pull request"**
6. Add description of changes
7. Click **"Create pull request"** again

### 4. Reviewing Code
1. Open a Pull Request
2. Click **"Files changed"** tab
3. Hover over line numbers to add comments
4. Click **"Review changes"** button
   - Comment
   - Approve
   - Request changes

### 5. Merging a Pull Request
1. Open the PR
2. Scroll to bottom
3. Click **"Merge pull request"**
4. Click **"Confirm merge"**
5. Optional: Click **"Delete branch"**

## Quick Navigation Tips
| Shortcut | What It Does |
|----------|--------------|
| `t` | File finder (search for files) |
| `s` or `/` | Focus search bar |
| `g` + `c` | Go to Code tab |
| `g` + `i` | Go to Issues |
| `g` + `p` | Go to Pull Requests |
| `.` | Open in github.dev (web editor) |
